#  1 健身表格完善
 后续会需要单独设计针对力量举周期计划的复杂计划表格，这个表格我想到的有这么几个要求：
 - 兼容力量举周期计划和普通增肌/自定义计划
- 展示层直观，一眼看出7天训练/休息安排
- 数据结构优雅，便于 AI 快速正确修改
- **模板与实例分离，支持多种组类型和次数表达方式，动作有主辅分级**

具体的健身计划展示层应该是类似课表,
一列的竖排, 从左到右边是周一到周日, 若是休息日的列就比较窄, 训练日的列就比较宽,然后一列内部有多个展示卡片, 一个卡片上记录了一个动作及其重量,组数及其他信息

# 2 一些复杂数值，比如TDEE，体脂率等的计算
这写复杂数值的计算不能依赖单纯的数学公式, 应该是AI依据记录的身体数据和训练记录, 力量数据, 等进行合理计算修正的结果.

# 3 UI设计
整体使用清亮的蓝紫色高亮+白色背景底板设计. 其颜色主题应该很像codex桌面app的设计. 我很喜欢这个codex的颜色设计.你按照这个设计来更新一下UI的颜色

# 4 数值显示
健身重量应该显示到小数点后两位, 比如卧推重量97.25这样.

# 5 健身计划页面的RPE数值错误
RPE应该只在0-10间, 不能超过10或者小于0

# 6 每日训练类型的修改
这里不能限制为推拉腿, 应该放开由用户自己输入, 后续agent获取信息时再读取字符串即可

# 7 deepseek API 优化
需要让AI详细阅读ds 文档后进行对应的各项优化,工具函数调用, 多轮对话,上下文拼接, 上下文磁盘缓存, 思考模式和思考强度设置等

# 8 Agent功能优化
- 上下文压缩, 上下文管理, 聊天记录保持等,只有20条记录太少了, 本地保存的聊天记录需要保存全部.
- token消耗优化, 去参考claude code和opencode等优秀开源agent的设计, 优化长对话, 多工具调用下的agent token消耗.
# 9 AI对话框 
- enter直接发送消息, shift+enter才是换行
- 对话框设计应该类似主流大模型网页端的设计,比如claude或者GPT的设计. 把对话框放到中间而不是旁边.
- 我需要能选择模型,pro模型还是flash模型.
- 模型输出是MD格式的文本. 应该要加载一个MD渲染器, 渲染MD格式, 而不是直接输出没有渲染的MD语法内容
- 支持文件上传解析主要是图片和excel, md文件和docx文件. 上传的文件保存在本地的一个缓存文件夹(不要C盘, 存在软件所在文件夹).
