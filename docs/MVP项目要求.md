# **一、课程报告基本内容要求**

## **1.1课题背景**

基于 AI Agent 课程中关于 VS Code、AI 辅助编程、Vibe Coding、SDD 与 TDD 的内容，完成 一个 Agent-based 的 MVP 应用，实现一项可落地、可演示、可验证的功能。整个项 目应以两种方式呈现：
1) 演讲环节：请准备幻灯片并在班级面前展示你们的项目。每个小组演讲时间不超过7 分钟，其中1-2分钟为问答环节。
2) 课程报告：以小组为单位撰写一份详细的报告，描述整个课程设计过程的过程

## **1.2报告要求**

1)   本项目是一个开放性命题，内容形式：
学生也可以自拟一个 Agent-based MVP 应用题目，围绕一个真实场景或具体痛点，使用 VS Code 与 AI 辅助编程工具完成 MVP 的需求分析、SDD、TDD、实现、验证与课堂 demo。该题目不要求项目规模很大，但必须能跑通一条完整的核心功能链路，并能说明 AI 在开发过程中的作用。

2)   学生请自行组建团队。每组学生需要完成适用于X分钟展示的幻灯片，包括提问1-2分钟。每位学生需要单独提交一份课程报告纸质版本，同组学生的报告雷同，但是首页的姓名和学号不雷同。同组学生获得相同的报告分数。

3)   学生需将小组成员分工及同伴评分的复印件附在纸质报告中。此同伴评分材料将作为老师评定最终成绩的参考。

		1）需求分析与项目背景；
		2）SDD 文档（idea.md / spec.md / plan.md / tasks.md）；
		3）TDD 或最小验证方案（测试用例、手动验收清单、截图或日志）； 
		4）AI 辅助编程过程与关键提示词；
		5）项目 demo 与功能链路展示；
		6）项目反思、局限与后续迭代方向；
		7）同行评分；
		8）任何你认为可以帮助改进报告的内 容

5)   请在报告正文中至少提供五个参考文献，可以是论文、网页或书籍。

6)   展示的评分标准包括完整性、逻辑性、展示技巧和合作；最终报告的评分标准包括完整性、逻辑性、准确性、课题的挑战性和同行评分。

## **1.4 Agent-based MVP** **应用作业整体流程：**

如果学生选择 Agent-based MVP 应用方向，请围绕一个真实问题自拟题目，例如：校园活动报名与答疑助手、论文阅读与摘要助手、实验室设备预约助手、课程资料问答助手、个人学习计划助手等。项目不要求使用复杂模型或云端部署，但必须体现“用 AI 辅助编程完成一个可运行 MVP”的完整过程。

**建议按照以下流程完成：**

需求分析：说明目标用户、使用场景、核心痛点、MVP 范围和不做的内容。题目可以自拟，但要明确解决什么问题。

轻量 SDD：至少提交 idea.md、spec.md、plan.md、tasks.md。idea.md 说明为什么做；spec.md 写清用户、功能、验收标准和边界；plan.md 说明技术方案、页面或模块结构、数据/API 草图和风险；tasks.md 将开发拆成 30–60 分钟可完成的小任务。

最小 TDD / 验证方案：不要求覆盖全部功能，但至少要设计 1 个核心成功路径、2–3 个边界情况和 1 个失败场景。可以使用自动化测试，也可以结合手动验收清单、运行截图、日志和预期输出。

AI 辅助编程过程：记录使用的工具（如 VS Code 插件、Cline、Cursor、Claude Code、Codex 或其他工具）、关键提示词、AI 生成内容、人工修改内容和遇到的问题。报告中必须保留若干条关键提示词，并说明这些提示词如何推动项目从计划、实现、调试到验证。

实现 MVP：至少完成一个可点击、可运行、可演示的核心功能链路。例如：首页/输入页 → 处理逻辑 → 结果页/管理页 → demo 展示页。项目可以采用本地 JSON、mock 数据或轻量前端方案，不强制接入真实数据库和真实大模型。

项目 demo：课堂展示时需要演示完整流程，并用 1 分钟左右说明“问题—方案—核心功能—验证结果—下一步迭代”。

提交材料：PPT、课程报告、源代码、运行说明、环境配置说明、关键提示词记录、测试或验收记录、必要截图。

**建议的 MVP 交付物包括：**

docs/spec.md、docs/plan.md、docs/tasks.md 或等价文档；

至少一个可运行功能；

关键提示词与 AI 辅助开发记录；

测试结果、手动验收清单或截图证明；

README.md，说明安装、运行、测试和 demo 步骤。
# **二、课程报告辅助资料**

## **2.5 Agent-based MVP作业整体流程：**

如果学生选择 Agent-based MVP 的内容形式，请按照课程教学中的“需求分析—SDD—TDD/验证—AI 辅助实现—DEMO”流程完成项目。项目最终需要完成一个完整的核心业务流程，能够在课堂上进行随堂 demo。报告中需要说明题目来源、目标用户、MVP 范围、工具链、关键提示词、代码实现过程、验证结果和后续迭代方向。注意：本题重点不是做一个“大而全”的系统，而是通过结构化 Vibe Coding，把一个小而明确的想法做成可运行、可验证、可展示的 MVP。

**MVP** **项目评分参考：满分 100 分**

如果学生选择 Agent-based MVP 项目，可参考以下标准评分。数据分析项目仍沿用上表评分标准。

演讲展示（15 分）：PPT 内容完整、页面美观、演示流程清晰、团队合作良好、时间控制合理。

需求分析与 SDD（20 分）：目标用户、使用场景、MVP 范围、验收标准和边界条件清晰；能够提交 idea/spec/plan/tasks 等结构化文档。

TDD 与验证记录（15 分）：有最小测试集、手动验收清单、截图、日志或预期输出；能够证明核心功能确实可用。

MVP 功能完成度（20 分）：完成至少一条完整功能链路；应用可以本地运行；交互流程和结果展示清楚。

AI 辅助编程过程与提示词（10 分）：能够记录关键提示词、说明 AI 生成和人工修改的边界，并总结调试过程。

创新性与挑战性（10 分）：选题有一定应用价值，能体现课程内容与个人研究/兴趣方向的结合。

代码完整度与说明文件（10 分）：代码完整、可运行，README 或说明文件清晰，包括环境配置、运行命令、测试命令和 demo 步骤。
