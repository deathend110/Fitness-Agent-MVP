# V2.7 AI 对话与 Agent 接口收口实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不破坏现有 AI 教练能力的前提下，统一 AI 对话、agent、计划卡和 provider runtime 的主链路接口，压薄兼容壳，并删除当前主流程之外的重复入口与无价值包装层。

**Architecture:** 本轮按“稳妥收口”推进，不做激进删除。前端统一到 `coachChat.js + coachBackend.js + backendClient.js`，后端统一到 `chat.py + chat_session.py + run_tool_calling_chat() + provider runtime`，旧接口先降级为兼容壳，再视测试与引用情况清理。

**Tech Stack:** React 19、Vite、FastAPI、SQLite、Node 内建测试、Pytest、OpenAI-compatible runtime、Gemini native runtime

---

## 一、文件与职责收口图

### 1. 当前收口后应保留为主入口的文件

- `src/tabs/CoachTab.jsx`
  - AI 教练页面状态协调入口
- `src/utils/coachChat.js`
  - AI 聊天请求语义层
- `src/api/coachBackend.js`
  - AI 聊天 HTTP 封装层
- `src/api/backendClient.js`
  - 普通 CRUD / 会话 / 草稿 / 文件 / 模型配置 / 计划卡操作封装层
- `backend/api/chat.py`
  - 后端 AI 主入口
- `backend/agent/chat_session.py`
  - agent 上下文与 provider runtime 接线主入口
- `backend/agent/tool_loop.py`
  - 协议无关工具回环
- `backend/agent/tool_calling.py`
  - 工具注册与执行校验
- `backend/agent/background_worker.py`
  - 后台思考任务主入口
- `backend/model_config/runtime.py`
  - 模型引用解析与 provider runtime 缓存

### 2. 本轮需要降权或清理的对象

- `src/api/deepseek.js`
  - 历史命名 API 包装层
- `backend/api/weekly_plan.py::POST /api/weekly-plan/adopt`
  - 旧版直接写回接口
- `backend/api/chat.py` 中旧 `messages` 契约的直接分支感知
  - 需要先统一归一再保留兼容
- `backend/agent/chat_session.py` 中带旧语义的 provider wrapper 命名
- 与主流程不一致的 README / ARCHITECTURE / backend README 表述

### 3. 本轮不动或只做注释性降权的对象

- `backend/agent/deepseek_client.py`
  - 暂不删除，只明确 fallback 身份
- 数据表结构
- 非 AI 页面的表单与展示组件
