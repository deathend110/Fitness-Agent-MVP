### Task 5.1 建立最小验证清单

- **估时**：45 分钟
- **依赖**：Task 4.6
- **目标**：满足课程对 TDD 或最小验证方案的要求。
- **主要操作**：
  - 在 `docs/verification.md` 中记录核心成功路径。
  - 记录 2-3 个边界情况。
  - 记录 1 个失败场景。
  - 每项写明前置数据、操作步骤、预期结果、实际结果。
- **完成信号**：
  - 文档能直接放进课程报告。
- **验证方式**：
  - 按文档从头操作一遍，确认步骤可复现。

### Task 5.2 补齐 README 和 ARCHITECTURE

- **估时**：45 分钟
- **依赖**：Task 4.6
- **目标**：保证助教可以运行和理解项目。
- **主要操作**：
  - 更新 `README.md`：安装、运行、API Key、测试、demo 步骤。
  - 更新 `ARCHITECTURE.md`：目录结构、模块职责、数据流、AI 调用链路。
  - 明确 localStorage 数据不会云端同步。
- **完成信号**：
  - 新同学按 README 能启动项目。
  - ARCHITECTURE 能说明核心闭环如何流动。
- **验证方式**：
  - 在干净环境中按 README 命令运行。

### Task 5.3 整理课堂 demo 脚本

- **估时**：30 分钟
- **依赖**：Task 5.1
- **目标**：保证 7 分钟展示有清晰路径。
- **主要操作**：
  - 在 `docs/demo_script.md` 中写 1 分钟项目介绍。
  - 写 3-4 分钟操作流程。
  - 写 1 分钟验证结果。
  - 写 1 分钟局限与后续迭代。
- **完成信号**：
  - demo 脚本覆盖“问题、方案、核心功能、验证结果、下一步迭代”。
- **验证方式**：
  - 按脚本计时演练一次，总时长不超过 7 分钟。
