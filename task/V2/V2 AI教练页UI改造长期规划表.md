# V2 AI 教练页 UI 重排长期规划表

## 范围说明

- 本轮只改 `AI 教练` tab 的 UI 重排，不改任何对话能力、模型接入、文件上传、草稿记忆、后台思考等功能逻辑。
- 本轮重点是把当前 AI 教练页改成更接近主流大模型网页端的聊天界面，整体参考 `Astrbot 聊天页面.png` 的布局语言。
- 当前页面里已有的提示词预览、最近日志摘要、右侧说明区等内容，不再作为这轮 UI 的主结构。
- 本轮不做 `V2.5` 之后的功能增强，也不改其他 tab。

## 现状结论

- 当前 `src/tabs/CoachTab.jsx` 还是偏“控制面板 + 预览面板”的三栏结构。
- 当前 `src/components/CoachConversationPanel.jsx` 仍是基础聊天框样式，输入框、消息区、发送区都比较朴素。
- 当前 AI 教练页虽然已经有对话能力，但页面布局还没有完全进入“聊天产品”的视觉状态。
- 所以这次更适合先做 UI 壳层与交互骨架重构，再把消息区、输入区和侧栏视觉统一起来。

## 目标

- 把 AI 教练页改成“中间主对话区 + 侧边辅助区”的聊天产品形态。
- 保留现有功能链路，但只调整视觉布局、信息层级和控件呈现方式。
- 页面视觉尽量贴近主流大模型网页端，而不是普通设置页或表单页。
