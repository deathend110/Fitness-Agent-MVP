# Task 1 AI 教练页全新聊天壳层重构

- **估时**：90 分钟
- **依赖**：无
- **目标**：把 AI 教练页彻底重写成一套全新的聊天页面，布局严格遵守 `Astrbot 聊天页面.png` 与 `Astrbot 聊天页面1.png` 的空间组织方式。

## 设计原则

- 这不是旧页面的局部修补，而是对 AI 教练页主结构的整体重写。
- 只保留与其他页面一致的主题颜色、边框、阴影与基础留白语言。
- 布局严格参考示意图的元素位置，不沿用当前 `CoachTab` 的三栏信息面板设计。
- 当前页面里原有的提示词预览、最近日志摘要、右侧信息面板，不再作为主界面结构。
- 不需要把旧页面的视觉骨架保留下来再叠加新样式，应该直接按新聊天页的层级重建。

## 参考界面

- 空状态参考：`Astrbot 聊天页面.png`
- 已对话状态参考：`Astrbot 聊天页面1.png`

### 空状态必须出现的结构

- 左侧浅色侧边栏。
- 侧边栏顶部返回箭头。
- 侧边栏顶部的“创建对话”入口。
- “项目”分组与“创建项目”入口。
- 聊天记录列表。
- 侧边栏底部“设置”入口。
- 主区域中央的欢迎文案。
- 中央的大输入卡片。
- 输入卡片左下角的 `+` 附件入口占位。
- 输入卡片中的模型标签占位。
- 输入卡片右侧的语音图标与发送图标占位。

### 已对话状态必须出现的结构

- 左侧浅色侧边栏继续固定显示。
- 侧边栏中的聊天记录列表需要可滚动。
- 主区域显示多轮消息流，用户与助手左右分布。
- 助手回复区域可带轻量操作图标占位。
- 页面右侧保留细长滚动条表现。
- 底部输入卡片始终贴在主内容区域下方，保持与空状态一致的核心形态。

## 数据与结构要求

- AI 教练页需要从“工具面板”重构为“聊天产品页面”。
- 左侧的聊天记录、项目入口、设置入口都属于页面主结构的一部分。
- 中央区域必须支持两种状态：
  1. 未开始对话时的欢迎空态
  2. 已有对话时的消息流状态
- 如果当前没有独立会话数据结构，也要先在 UI 层预留“会话列表 / 当前会话”的表达方式，不能再用旧的三栏辅助面板去充当聊天记录。
- 现有 `chatHistory` 可以作为消息流输入，但页面的视觉承载方式必须按照新聊天页重排。

## 任务拆分

### 子任务 1.1：重建页面壳层与左侧栏

- [ ] 设计新的 AI 教练页整体壳层，改成“左侧栏 + 主聊天区域”的二栏结构。
- [ ] 左侧栏按示意图组织：返回、创建对话、项目分组、聊天记录、设置入口。
- [ ] 去掉旧页面里三栏信息面板的主导感，不再让提示词预览或日志摘要占据主视觉。

### 子任务 1.2：重建空状态聊天页

- [ ] 主区域空状态显示欢迎语。
- [ ] 中央输入卡片严格按示意图排版，输入框为视觉中心。
- [ ] `+`、模型标签、语音、发送图标都按图预留在输入卡片内。
- [ ] 空状态下不显示旧的辅助卡片堆叠。

### 子任务 1.3：重建已对话状态

- [ ] 对话开始后，主区域切换为消息流视图。
- [ ] 用户消息右对齐，助手消息左对齐。
- [ ] 长对话支持纵向滚动，滚动条位置和视觉样式贴近示意图。
- [ ] 输入卡片保留在底部，保持与空状态一致的核心结构。

### 子任务 1.4：收口与旧结构清理

- [ ] 检查旧的提示词预览、日志摘要、右侧辅助面板是否已从主结构移除。
- [ ] 确认页面主题颜色与其他 tab 保持一致，只调整布局，不换风格。
- [ ] 检查在 16:9 全屏下是否不会出现横向滚动。
- [ ] 为后续任务保留最少必要的组件边界，不要把旧设计重新包回来。

- 发送一条消息后，确认页面切换到已对话状态，消息布局与滚动行为正常。
- 检查左侧记录列表、项目区、设置入口是否都在正确位置。
- 检查 16:9 全屏下不会出现横向滚动。
- 检查页面没有恢复成旧的三栏工具面板样式。
