# 一、任务拆分原则

- 每个任务控制在 30-60 分钟内完成。
- 任务按依赖顺序推进，前置任务没有完成时不开始后置任务。
- 每个任务完成后都要能看到明确结果，例如文件生成、页面可见、数据可保存、错误可提示。
- 遵守 `plan -> code -> review`：每次进入新任务前先确认目标，完成后做最小验证和自查。
- 代码文件超过 200 行时必须评估拆分，优先拆到 `components/`、`utils/` 或 `api/`。
- 每次代码更新后同步维护 `README.md` 和 `ARCHITECTURE.md`。

## 二、总体依赖顺序

```
项目骨架
  -> 基础工具与默认数据
  -> 4 Tab 页面框架
  -> 档案录入
  -> 训练计划展示
  -> 训练计划编辑
  -> 今日日志
  -> Prompt 构建
  -> AI 对话
  -> JSON 建议解析
  -> 一键采纳写回
  -> 验证与演示材料
```

核心 demo 必须跑通：

```
填写档案 -> 设置训练计划 -> 录入今日日志 -> 询问 AI -> 渲染建议卡片 -> 采纳并更新训练计划
```
