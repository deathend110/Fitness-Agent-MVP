# V2.3 Phase 3 验收记录

> 日期：2026-05-31
> 范围：Task 13-21，上下文管理、长期 memory、token/cache 观测、DeepSeek Tool Calls、计划 propose/commit 与前端 Agent 请求契约。

## 验收范围

本阶段把 Phase 2 的后端聊天代理升级为后端 Agent Orchestrator：

- 后端统一构建 stable system prompt 与动态上下文。
- 长对话超过预算时进行摘要压缩，并回注当前状态。
- 记录 DeepSeek token usage 与 prompt cache hit/miss。
- 提取、检索、确认长期 memory，安全类记忆优先注入。
- 接入 DeepSeek Tool Calls，只读工具可读取档案、周计划、今日日志、轻量指标和 memory。
- 计划修改采用 propose/commit 两段式，模型只能生成建议卡，用户确认后才写库。
- 前端 AI 教练发送链路改为 `sessionId + userInput + model/thinking + files`，不再构建 system prompt。

不在本阶段范围内：

- 真实多会话 UI。
- 文件上传解析与上传文件全文工具。
- 向量知识库。
- 完整 Python 指标服务。
- 周期计划引擎与预制计划库。

## 手动 Demo 验收路径

### 核心成功路径

1. 启动 `npm run dev:all`。
2. 在“我的档案”填写基础信息、目标和三大项 1RM。
3. 在“训练计划”安排至少一个包含深蹲或卧推的训练日。
4. 在“今日日志”填写体重、热量、睡眠、疲劳和训练备注。
5. 进入“AI 教练”，发送：“根据我这周深蹲表现和今天疲劳，帮我调整明天训练。”
6. 后端从 SQLite 读取 profile / weeklyPlan / dailyLog / memory / recent messages，必要时调用只读工具。
7. AI 返回解释与计划修改建议卡。
8. 用户点击“采纳并更新计划”。
9. 后端通过 `/api/tools/plan/commit` 校验 proposal 并写回 `weekly_plan_day.exercises`。
10. 刷新页面，训练计划仍保留更新结果。

### 长对话边界

1. 构造 80+ 轮聊天历史或使用测试 fixture。
2. 再发送新的训练调整问题。
3. 后端触发 `SummaryCompressor` 或复用已有 `ChatSessionSummary`。
4. 新上下文仍包含安全限制、当前周计划、今日日志、近期用户输入和未确认 proposal。
5. `/api/chat/sessions/{id}/context/debug` 可看到估算 token、摘要选择和 usage 汇总。

### Memory 边界

1. 用户明确说：“我膝盖深蹲到底部会疼。”
2. 该信息进入 `kind=safety` 的 memory 候选或确认项，并在后续上下文中优先注入。
3. 用户只说“今天很累”时，不应自动晋升为长期 memory。
4. 低置信度候选通过 `/api/memory/candidates/{id}/confirm` 晋升，或通过 `/ignore` 忽略。

### 工具与计划边界

1. 模型请求不存在的动作名、非法 day、未知字段、非有限数字或越界 RPE。
2. 后端 Pydantic/adopt 校验拒绝请求。
3. 工具返回可解释错误，`ToolCallLog` 记录失败。
4. 周计划保持原状，不写入脏数据。

### 失败场景

- DeepSeek API Key 缺失、上游超时或返回错误：SSE 返回 `error`，后端不保存半截 assistant。
- 工具循环超过 4 轮：返回友好错误并记录日志，不继续无限调用。
- 摘要压缩连续失败：进入熔断，保留安全记忆、当前计划和最近窗口。
- proposal 重复提交、过期或找不到：commit 返回失败，计划不变。
