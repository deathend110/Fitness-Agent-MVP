# V2.3 Phase 1 验收记录

## 范围

本次验收对应：

- Task 1：后端工程初始化与本地启动编排
- Task 2：SQLite 数据层与 ORM 模型
- Task 3：档案 / 周计划 / 日志 CRUD 接口

不在本次范围内：

- `chatHistory` 后端化

- 计划采纳后端化

## 完成结果

### 已完成能力

- 新建 `backend/` 本地 Python 服务
- 建立 SQLite + SQLAlchemy Async 数据层
- 建立 Alembic 初始化迁移
- 提供 `profile / weeklyPlan / dailyLog` 三组后端 CRUD
- 前端 `App.jsx` 已切到后端优先加载主数据
- 前端仍保持 `profile.oneRM` 与 `dailyLog[date].tdee` 的内部数据形状
- 提供 `POST /api/migrate/import`
- 启动时可检测“本地有旧数据且后端为空库”并提示一键导入
- 档案页的数据备份区支持手动“导入到后端”

### 当前保留行为

- `chatHistory` 继续只存在 localStorage
- 导入接口接收 `chatHistory` 但明确跳过入库
- 后端不可用时前端回退到本地缓存展示，并提示降级状态

## 手动验收路径

### 核心成功路径

1. 启动 `npm run dev:all`
2. 打开页面，确认顶部显示“已切换为后端数据源”或明确的降级提示
3. 在“我的档案”填写基础信息与 `1RM`
4. 在“训练计划”修改某一天的训练类型或动作
5. 在“今日日志”保存当天记录
6. 刷新页面，确认三类数据仍然保留

### 边界情况

1. 后端空库启动：
   - 页面应进入空白默认态
   - 不应白屏
2. 本地已有旧数据、后端为空库：
   - 页面应出现一键导入提示
   - 点击后应把 `profile / weeklyPlan / dailyLog` 导入后端
3. 重复导入同一份备份：
   - 数据不应重复
   - 周计划仍应只有 7 天

### 失败场景

1. 后端未启动：
   - 页面应显示后端不可用的降级提示
   - 档案页“导入到后端”按钮应禁用
   - 本地缓存页面仍可继续展示

## 本阶段遗留

- `chatHistory` 仍未迁移到数据库
- 前端 AI 仍未经过后端代理
- 还没有 SSE 流式与后台思考
- 计划采纳仍未迁到后端校验
