# V2.3 Phase 4 开发完成报告

## 目标回顾

Phase 4 的目标是把 AI 教练从“后端 Agent 能跑通”推进到“演示体验顺手”：用户可以上传训练资料、选择模型和 thinking、看到结构化 Markdown 回复、刷新后保留输入草稿，并让复杂指标逐步迁到 Python 后端统一口径。

## 完成内容

### 1. 文件上传与解析

- 新增 `UploadedFile` 与本地缓存目录 `backend/data/uploads`。
- 支持 Markdown/TXT、DOCX、Excel 和图片的轻量解析。
- 解析摘要统一为 `{kind,title,summary,preview,text}`，进入 prompt 前只注入裁剪摘要。
- 可解析文本同步写入 `KnowledgeItem(kind="uploaded_file")`，保留来源文件 id。
- 上传限制覆盖大小、扩展名、空文件、解析失败和 metadata 读取/删除。

### 2. 文件上下文与工具读取

- 聊天请求从前端传 `fileIds`，不传本地路径或完整文件内容。
- `build_agent_request()` 读取 `UploadedFile.summary` 并注入 `PromptAssembler`。
- debug metadata 增加 `selected_files / missing_files / trimmed_file_summaries`。
- `read_uploaded_file_summary` 从占位工具升级为真实读取上传文件摘要。

### 3. 模型列表与 Thinking 配置

- 新增 `GET /api/models`，优先请求 DeepSeek `/models` 并按 `MODEL_ALLOWLIST` 过滤。
- 缺 Key 或上游失败时返回 fallback 白名单、`source` 和 `warning`。
- 前端新增 `ModelSelector`，发送聊天时携带 `model/thinking`。
- 收尾审查补齐 DeepSeek Thinking 透传：后端将 `{enabled,budget}` 映射为 DeepSeek `thinking` 与 `reasoning_effort`，思考模式工具轮次保留 `reasoning_content`，并避免发送不兼容的 `tool_choice`。
- 修复模型列表慢回包覆盖后端草稿中已恢复模型/Thinking 的竞态。

### 4. Markdown 与滚动体验

- 新增安全 Markdown parser 与 `MarkdownMessage`。
- 支持标题、列表、加粗、行内代码、代码块和安全链接。
- 不使用 `dangerouslySetInnerHTML`；危险链接和 HTML 降级处理。
- `MessageList` 新增底部 sentinel，对新消息、流式输出和后台结果合并滚动到最新。

### 5. Coach 草稿持久化与附件 UI

- 新增 `CoachDraft` 与 `GET/PUT /api/chat/sessions/{id}/draft`。
- 草稿保存输入内容、模型、thinking 和附件 id；未知附件 id 返回 422。
- 新增 `FileAttachmentTray`，支持上传状态、附件 chip 和删除。
- 发送成功后清空草稿内容和附件，保留模型配置。

### 6. Python 复杂指标服务

- 新增 `backend/metrics/daily_metrics.py` 与 `GET /api/metrics/daily-summary`。
- 热量状态规则统一为 `delta > 100 => surplus`、`delta < -100 => deficit`、其余 `balanced`。
- 蛋白质规则统一为 `protein_g_per_kg >= 1.6 => met`，否则 `low`。
- 前端 prompt/指标测试已校准，避免展示和 prompt 口径漂移。

## 已知边界

- AI 教练历史侧栏仍是当前前端模型，真实多会话 UI 留 Phase 5。
- 上传图片当前只解析 metadata，不做 OCR 或视觉模型读图。
- 文件摘要使用关键词/裁剪注入，不是向量知识库。
- Python 指标服务已对齐当前 MVP 口径，但 AI 修正 TDEE/体脂率持久化留后续。
- 周期计划引擎和预制计划库未进入 Phase 4。

## 下一阶段建议

Phase 5 建议优先顺序：

1. 真实多会话 UI：前端侧栏接后端 `chat_session`，刷新后按会话拉取消息。
2. 知识库增强：基于 `KnowledgeItem` 和 `UploadedFile` 做跨对话检索，评估 sqlite-vec 或轻量向量库。
3. 周期计划引擎：引入周期状态、预制计划库和按 1RM 自动填重。
4. 指标深化：把 AI 修正 TDEE/体脂率和周期训练负荷纳入后端指标体系。
