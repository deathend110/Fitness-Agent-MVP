## 边界情况

- 超大文件：超过 `MAX_UPLOAD_MB` 返回 413，不写脏 DB。
- 不支持格式：如 `.exe` 返回 415。
- 空文件：返回 422，不写文件记录。
- 文件解析失败：保留 `UploadedFile` 和 `parser_error`，不阻断普通聊天。
- 缺少 DeepSeek API Key 或 `/models` 上游失败：`/api/models` 返回本地 fallback 与 warning。
- 草稿保存未知附件 id：返回 422，避免 UI 误以为附件已恢复。
- Markdown 原始 HTML 和危险链接：按文本或安全链接处理，不使用 `dangerouslySetInnerHTML`。
- 指标缺少身高/体重/日志：返回稳定 unknown/null 字段，不 500。

## 失败场景

- 文件解析失败时，用户仍可删除附件或继续无附件聊天。
- 模型列表失败时，前端使用后端 fallback 默认模型，不阻断聊天。
- 草稿保存失败时，前端保留本地输入态，避免用户正在编辑的内容丢失。
- DeepSeek 流式失败仍沿用 Phase 2/3 的非流式回退和不写半截 assistant 规则。
