# V2.3 开发完成总结

> 日期：2026-05-31
> 更新：2026-06-01
> 分支：`feature/v2.3-python-backend`，已合入当前 `master`
> 范围：Phase 1 后端骨架与数据迁移 + Phase 2 Python Agent 与密钥后移 + Phase 3 上下文管理与工具调用 + Phase 4 文件体验与模型配置

## 完成范围

### Phase 1：本地 Python 后端基建

- 新增 `backend/` FastAPI 服务、SQLite 数据层、SQLAlchemy async session 与 Alembic 迁移。
- 将 `profile / weeklyPlan / dailyLog` 接入后端 CRUD。
- 前端主数据加载与保存改为后端优先，后端不可用时保留本地缓存降级提示。
- 提供 `localStorage -> SQLite` 一次性导入接口；旧版 `chatHistory` 在迁移接口中明确跳过，后续由 chat 表承接。

### Phase 2：Python Agent 与密钥后移

- DeepSeek HTTP 客户端迁到后端，API Key 只由 `backend/.env` 读取，前端不再直连 DeepSeek。
- `---JSON---` 响应解析迁到 `backend/agent/response_parser.py`，后端统一拆分文本与结构化 `suggestion`。
- 新增 `ChatSession / ChatMessage`，支持默认会话、全量消息读取和 `suggestion` JSON 存储。
- 新增 `/api/chat/stream` SSE 代理，事件为 `delta / suggestion / done / error`；成功后写入 user + assistant。
- 新增 `/api/chat/reply` 非流式回退接口。
- 新增离页后台思考：`POST /api/chat/{session_id}/background` 提交任务，`GET /api/chat/background/{task_id}` 查询结果；失败任务不写脏 assistant。
- 前端 AI 教练页接入后端 SSE；离页时中止前台请求并提交后台任务，回页后校验源 user 消息下标与内容再补 assistant，避免旧后台结果污染新对话或同文本重开对话。
- 新增 `/api/weekly-plan/adopt`，将计划采纳校验迁到后端；只支持显式 `action: "update"`，失败不写库，成功返回完整 plan。
- 采纳校验支持 `pct / kg / sets / reps / rpe` 的安全数字转换，拒绝 `NaN / Infinity` 等非有限数值；更新 `reps` 时同步 `template.repsText`。

### Phase 3：上下文管理与工具调用

- 新增 `chat_session_summary / memory_item / knowledge_item / tool_call_log / usage_record` 五类表，支撑摘要、长期记忆、知识、工具审计和 token/cache 观测。
- 新增 `PromptAssembler`，由后端统一按稳定 system prompt、当前状态、安全记忆、相关记忆、摘要、最近消息和当前输入拼装 DeepSeek messages。
- 新增 `SummaryCompressor` 与 `StateReinjector`，长对话超预算时可写入摘要，并在压缩后重新注入当前计划、今日日志和待确认状态。
- 新增 `UsageLedger`，记录 DeepSeek `prompt_tokens / completion_tokens / total_tokens / prompt_cache_hit_tokens / prompt_cache_miss_tokens`，并通过 `/api/chat/sessions/{id}/context/debug` 调试。
- 新增长期 memory 提取、检索、候选确认和忽略接口；安全类记忆优先注入，临时单日状态不会自动晋升。
- 新增 DeepSeek Tool Calls 基础设施、只读工具、工具结果瘦身和工具调用日志；工具参数使用 strict schema + Pydantic 双重校验。
- 新增计划修改 `propose/commit` 两段式接口：模型只能生成建议卡，用户确认后才写回训练计划。
- 前端 AI 教练发送链路切到后端 Agent Orchestrator，默认只提交 `sessionId + userInput + model/thinking + files`，前端不再构建 system prompt。

### Phase 4：文件体验、模型配置、Markdown 与草稿持久化

- 新增 `UploadedFile / CoachDraft` 数据模型与 `backend/data/uploads` 本地文件缓存目录。
- 支持 Markdown/TXT、DOCX、Excel 和图片文件上传；可解析内容生成轻量摘要，进入 `UploadedFile`，文本资料同步进入 `KnowledgeItem(kind="uploaded_file")`。
- 聊天请求支持传入 `fileIds`，后端按摘要注入 Agent 上下文；`read_uploaded_file_summary` 工具从占位实现升级为读取真实上传文件摘要。
- 新增 `GET /api/models`，由后端统一读取 DeepSeek 模型列表；缺 Key 或上游失败时返回本地 fallback、`source` 和 `warning`。
- 前端 AI 教练页新增模型与 Thinking 配置选择，发送请求携带 `model/thinking`；后端负责映射 DeepSeek thinking / reasoning 参数。
- 新增安全 Markdown 渲染，支持标题、列表、加粗、行内代码、代码块和安全链接，不使用 `dangerouslySetInnerHTML`。
- 新增对话自动滚动到底部，覆盖新消息、流式输出和后台结果合并场景。
- 新增 Coach 草稿后端持久化，保存输入内容、模型、thinking 与附件 id；刷新后可恢复未发送草稿。
- 新增 Python 复杂指标服务 MVP：`GET /api/metrics/daily-summary`，并与前端 `buildDailyMetricsSummary()` 口径校准。

## 关键验收点

- 成功路径：用户发送 AI 消息 -> 后端 SSE 流式返回 -> 完整回复落库 -> 结构化建议可由用户点击采纳 -> 后端校验并写回周计划。
- 边界情况：聊天消息全量读取不裁剪；后台任务并发隔离；后台结果回页前校验源 user 消息；采纳批量 changes；字符串数字可安全转为数值。
- 失败场景：DeepSeek 错误通过 SSE `error` 返回且不写半截 assistant；后台空回复进入 `failed` 且不写库；非法 action、未知动作、未知字段、越界 RPE、非法数字和非有限数值均拒绝并保留原计划。
- Phase 3 成功路径：前端发送用户输入 -> 后端读取状态并组装上下文 -> 必要时调用只读工具 -> 返回文本与计划 proposal -> 用户确认 -> commit 写回周计划。
- Phase 3 边界情况：长对话摘要压缩后仍保留安全限制和当前计划；memory 只晋升稳定事实；工具参数非法时返回错误并不写计划。
- Phase 3 失败场景：DeepSeek 上游错误、工具循环超限、摘要压缩失败、proposal 重复提交或过期都不会保存半截 assistant 或写入脏计划。
- Phase 4 成功路径：用户上传资料 -> 前端只传 `fileIds` -> 后端注入摘要上下文 -> AI 回复按 Markdown 安全渲染 -> 草稿刷新后恢复。
- Phase 4 边界情况：模型列表失败返回 fallback；草稿未知附件 id 返回 422；文件解析失败保留错误状态且不阻断普通聊天。
- Phase 4 失败场景：超大文件返回 413，不支持格式返回 415，空文件返回 422；危险 Markdown 链接和 HTML 不会作为可执行内容渲染。

## 遗留与后续

- AI 教练页真实多会话 UI：后端已有会话与消息表，但前端展示仍主要依赖 `fitloop_chatHistory`。
- 文件上传解析、模型选择、Markdown 渲染、草稿持久化和 Python 指标服务 MVP 已在 Phase 4 完成；上传文件全文知识库、OCR/视觉理解和向量检索留 Phase 5 / V3。
- Memory 当前为规则/关键词 MVP，未接向量检索；Python 指标服务已完成 MVP，AI 修正 TDEE/体脂率持久化和周期训练负荷指标留后续。
- proposal 存储当前为进程内 MVP，服务重启后未提交的 proposal 会失效；如需跨重启保存，后续可持久化到数据库。
- 周期计划、预制计划库和跨周期自动递进留 V3。
- 后台任务当前为进程内内存表，服务重启后旧 `task_id` 会返回 `not_found`；后续可按需要引入持久化任务表或幂等 turn id。
