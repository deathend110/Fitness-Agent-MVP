## 关键架构决策（落地前必须对齐）

1. **不搬 `frontend/` 目录**：前端继续留在仓库根 `src/`，`backend/` 作为兄弟目录新增，降低本轮 churn；设计文档第六节的 `frontend/` 重组留到后续版本，可选。
2. **数据契约零改写**：`weekly_plan_day.day_key` 用全名星期键 `Monday..Sunday`（与现有代码一致，非 `Mon..Sun`）；`exercises` 整存 **JSON 列**，原样保留 `template + instance + 扁平兼容字段`；`profile / dailyLog` 形状不变；统一用 pydantic schema 固化契约。
3. **Phase 3 后端升级为 Agent Orchestrator**：前端不再构建 system prompt，也不再发送完整 `messages[]` 作为事实来源；后端从 SQLite 读取主数据、聊天、摘要、memory、knowledge，自行组装 DeepSeek `messages` 并执行工具循环。
4. **DeepSeek 适配优先**：Claude Code 分析只借鉴上下文治理思想；具体参数、Thinking Mode、Tool Calls、Context Caching、usage cache hit/miss 必须按 DeepSeek 官方 API 文档实现。
5. **工具调用安全边界**：模型只能调用只读工具和 `propose_plan_change`；真正写计划必须经过前端用户确认按钮，再走后端 commit/adopt 校验。
6. **测试策略**：前端继续用 Node 内置 `node:test`（不引 DOM）；后端使用 `pytest + pytest-asyncio`。每个子任务验证 = 定向测试（pytest 或 node --test）+ `npm run build` + 必要的手动联调。
7. **密钥纪律**：`DEEPSEEK_API_KEY` 只存 `backend/.env`，前端永不接触；缺失时后端接口返回明确错误、前端给友好降级。`backend/.env` 与 `backend/data/`（repmind.db + 未来 uploads）进 `.gitignore`。

## 风险提醒

- **数据迁移丢失**：导入接口必须幂等可重试；迁移前自动 JSON 备份（复用 `buildBackupPayload`）；导入后保留 localStorage 原值直到用户确认。
- **前后端契约漂移**：pydantic schema 固化 + `exercises` JSON 往返测试；保持现有数据形状不变。
- **本地双进程启动复杂**：根目录加 `npm run dev:all`（`concurrently` 同起 Vite + uvicorn），README 写清。
- **SSE 断连**：前端流式失败回退后端非流式接口；离页用后台任务兜底，回页拉取最新结果。
- **范围蔓延**：Phase 3 只做 Agent 编排、上下文、memory、token/cache 观测和计划工具；模型选择 / 文件上传 / MD 渲染 / 草稿持久化 / 周期计划仍留 Phase 4–5。
- **Prompt 前缀不稳定导致缓存失效**：system prompt 与工具 schema 要稳定前置；实时指标、日期、用户输入、工具结果后置；Task 15 必须记录 DeepSeek cache hit/miss。
- **Memory 污染**：单日状态不得自动晋升长期 memory；伤病/疼痛等 safety memory 需要来源和置信度；低置信度候选让用户确认。
- **工具误写计划**：模型只允许 propose，不允许直接 commit；commit 必须来自用户确认按钮，并复用后端 adopt 校验。
- **工具循环失控**：每轮工具调用次数上限 4 次；超限返回友好错误，不保存半截 assistant。
