# V2.3 Phase 2 Python Agent 与密钥后移实施计划

> **For agentic workers:** REQUIRED SUB-SKILL：使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按任务执行。步骤用 checkbox 语法追踪。
> 上游：[V2.3 Python 后端基建长期规划表](<./V2.3 Python 后端基建长期规划表.md>) / [docs/RepMind Python 重构设计.md](<../../docs/RepMind Python 重构设计.md>)
> 前置：Phase 1 必须全绿（CRUD + 前端切源 + 迁移可用）。详见 [V2.3 Phase 1 后端骨架与数据迁移实施计划](<./V2.3 Phase 1 后端骨架与数据迁移实施计划.md>)。

**Goal:** 把 DeepSeek 调用从浏览器直连改为后端代理，API Key 只存后端 `.env`；`chat` 全量落库（去掉 20 条裁剪）；提供 SSE 流式回复与离页后台思考；计划采纳校验后端化。

**Architecture:** Phase 2 后端是「瘦聊天代理」——**system prompt 仍由前端构建**（复用现有 [prompt.js](<../../src/utils/prompt.js>) 与指标单一口径 `buildDailyMetricsSummary`），前端把 `messages[]` 发给后端，后端负责藏密钥、调 DeepSeek、SSE 流式、消息落库、后台任务。prompt / 指标 / 上下文管理迁移属 Phase 3，不在本轮。`backend/agent/` 迁移 [deepseek.js](<../../src/api/deepseek.js>)（HTTP 客户端 + 错误码）与 [aiResponse.js](<../../src/utils/aiResponse.js>)（`---JSON---` 解析）；[adoptPlan.js](<../../src/utils/adoptPlan.js>) 的校验语义搬到后端服务 `/adopt`。chat 先用「默认会话」承接现有单条 `chatHistory`，真实多会话 UI 留 Phase 5 / V2.5。

**Tech Stack:** 续 Phase 1（FastAPI / SQLAlchemy async / pytest）；新增 httpx 异步流式、`StreamingResponse`（`text/event-stream`）、asyncio 后台任务。前端用 `fetch` ReadableStream / `EventSource` 消费 SSE。
