# V2.3 Phase 1 后端骨架与数据迁移实施计划

> **For agentic workers:** REQUIRED SUB-SKILL：使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按任务执行。步骤用 checkbox 语法追踪。
> 上游：[V2.3 Python 后端基建长期规划表](<./V2.3 Python 后端基建长期规划表.md>) / [docs/RepMind Python 重构设计.md](<../../docs/RepMind Python 重构设计.md>)

**Goal:** 新建本地 Python 后端（FastAPI + SQLite），把 `profile / weeklyPlan / dailyLog` 三类非 AI 数据迁到后端 CRUD，前端从直读 localStorage 改为调后端；提供 localStorage→DB 一次性导入与后端不可用降级。`chatHistory` 本阶段暂留 localStorage，Phase 2 再随 Agent 迁移。

**Architecture:** `backend/` 作为仓库根的兄弟目录新增，前端继续留在 `src/`。后端按 `db / api` 分层：`db` 管引擎/ORM/迁移/seed，`api` 管路由 + pydantic 契约。数据契约严格沿用现状——`weekly_plan_day.day_key` 用全名星期键 `Monday..Sunday`，`exercises` 整存 JSON 列保留 `template+instance+扁平字段`，`profile/dailyLog` 形状不变。前端在 [src/App.jsx](<../../src/App.jsx>) 这一唯一状态源把读写换成后端调用。

**Tech Stack:** Python 3.11+、FastAPI、uvicorn、SQLAlchemy 2.x（async）、aiosqlite、Alembic、pydantic-settings、httpx；后端测试用 pytest + pytest-asyncio。前端沿用 React 19 / Vite / Node 内置测试。
