# V2.3 Phase 4 文件体验与模型配置实施计划

> **For agentic workers:** REQUIRED SUB-SKILL：使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按任务执行。步骤用 checkbox 语法追踪。
> 上游：[V2.3 Python 后端基建长期规划表](<./V2.3 Python 后端基建长期规划表.md>) / [docs/RepMind Python 重构设计.md](<../../docs/RepMind Python 重构设计.md>)
> 前置：Phase 1 + Phase 2 + Phase 3 必须全绿。当前已具备 FastAPI + SQLite、主数据 CRUD、chat 全量存储、DeepSeek 后端代理、SSE、离页后台思考、后端 Agent Orchestrator、上下文压缩、memory、Tool Calls 和计划 propose/commit。

**Goal:** 补齐 Phase 4 的用户体验能力：文件上传解析、模型选择与 thinking 配置、Markdown 回复渲染、草稿后端持久化、对话滚动定位到最新，并把复杂指标计算迁到 Python MVP，确保 AI 教练从“能用”升级到“顺手可演示”。

**Architecture:** Phase 4 不推翻 Phase 3 Agent Orchestrator，而是在其外侧补用户体验层。文件上传进入 `backend/data/uploads/`，解析结果进入 `uploaded_file / knowledge_item`，聊天请求只传 `fileIds` 并由后端按摘要注入上下文。模型列表由后端统一管理：优先请求 DeepSeek `/models`，失败时回退本地白名单；前端只渲染后端返回的模型、thinking 选项和默认值。前端 Markdown、草稿和滚动属于 Coach UI 体验增强，保持业务写入边界不变。

**Tech Stack:** 续 Phase 1-3（FastAPI / SQLAlchemy async / pytest / httpx / SSE / React / Vite / Node test）。新增 Python 依赖：`python-multipart`、`pillow`、`openpyxl`、`python-docx`。Markdown 渲染优先使用前端小型安全渲染组件，不引入危险 HTML；如后续引入第三方 Markdown 包，必须先补安全测试。

**External References:**
- DeepSeek `/models`：官方接口 `GET https://api.deepseek.com/models` 返回可用模型列表，示例含 `deepseek-v4-flash` 与 `deepseek-v4-pro`。
- DeepSeek 模型与 pricing：当前 V4 Flash/Pro 支持 thinking、JSON Output、Tool Calls；`deepseek-chat / deepseek-reasoner` 为兼容名并将在 2026-07-24 退役。
- DeepSeek Thinking Mode：thinking 参数由后端透传；带工具调用时必须按官方语义处理后续轮次所需的 `reasoning_content`。

## 范围与非目标

本阶段必须完成：

- 文件上传：支持图片、Excel、DOCX、Markdown/文本文件，保存到 `backend/data/uploads/`，生成轻量摘要和 `fileId`。
- 文件上下文：AI 教练可携带 `fileIds`，后端注入摘要；工具 `read_uploaded_file_summary` 从占位升级为读取真实文件摘要。
- 模型选择：`GET /api/models` 返回后端白名单、远端可用模型、默认模型和 thinking 配置；前端输入区旁可选择模型和思考模式。
- Markdown 渲染：assistant 回复支持标题、列表、加粗、代码块、行内代码、链接文本的安全渲染。
- 草稿持久化：Coach 输入草稿、模型选择和 thinking 配置后端保存，切页/刷新后恢复。
- 滚动体验：对话进入、发送、流式输出和后台回复完成时自动定位到最新消息。
- 复杂指标 Python MVP：后端提供与当前 `buildDailyMetricsSummary()` 对齐的指标摘要接口，为后续完全后端化打底，并修复前端 prompt/指标口径漂移。
- 文档、验收记录和开发完成报告。

本阶段不做：

- 向量知识库。
- 真实多会话 UI 侧栏替换。
- 周期计划引擎和预制计划库。
- 云同步、账号系统、远端文件存储。
- 让模型直接写计划；计划写回仍必须经过用户确认。
