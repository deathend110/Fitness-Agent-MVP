
### Task 3.1 实现今日日志表单

- **估时**：45 分钟
- **依赖**：Task 1.2
- **目标**：用户能记录今天的体重、饮食和训练状态。
- **主要操作**：
  - 在 `TodayTab.jsx` 中读取 `getTodayStr()`。
  - 表单字段包括体重、热量、蛋白质、睡眠、疲劳度、是否完成训练、训练备注。
  - 保存到 `fitloop_dailyLog[today]`。
- **完成信号**：
  - 刷新页面后，今天的日志仍保留。
  - 未填写的可选字段不导致页面崩溃。
- **验证方式**：
  - 录入“体重 82kg，热量 2100kcal，疲劳度 3，备注：深蹲第3组没完成”，刷新后检查仍存在。

### Task 3.2 展示今日计划只读摘要

- **估时**：30 分钟
- **依赖**：Task 2.4、Task 3.1
- **目标**：用户录日志时能对照今天训练计划。
- **主要操作**：
  - 在 `TodayTab.jsx` 中读取 `weeklyPlan[todayKey]`。
  - 只读展示今日训练类型和动作摘要。
  - 休息日显示“今天是休息日”。
- **完成信号**：
  - 训练日显示动作列表。
  - 休息日不显示空白异常。
- **验证方式**：
  - 分别把今天设置为训练日和 rest，检查文案变化。

### Task 3.3 实现 System Prompt 构建

- **估时**：60 分钟
- **依赖**：Task 1.3、Task 2.4、Task 3.1
- **目标**：把最新用户状态打包给 AI。
- **主要操作**：
  - 创建 `src/utils/prompt.js`。
  - 实现 `buildSystemPrompt(profile, weeklyPlan, dailyLog)`。
  - 包含基本信息、三大项 1RM、本周训练计划、近 14 天体重、近 7 天饮食、近 7 天训练情况、今日 TDEE。
  - 没有数据时使用“暂无记录”或“未记录”。
- **完成信号**：
  - prompt 文本包含用户最新档案、计划、日志。
  - 新用户空日志不会报错。
- **验证方式**：
  - 在 AI 教练页临时输出上下文预览，检查是否包含“深蹲第3组没完成”和 TDEE 数据。
