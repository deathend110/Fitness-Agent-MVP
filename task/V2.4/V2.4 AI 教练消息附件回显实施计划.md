# V2.4 AI 教练消息附件回显实施计划

> **For agentic workers:** REQUIRED SUB-SKILL：使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按任务执行。步骤用 checkbox 语法追踪。
> 上游设计：[V2.4 AI 教练消息附件回显设计.md](<./V2.4 AI 教练消息附件回显设计.md>)
> 参考实现：[V2.3 Phase 4 文件体验与模型配置实施计划.md](<../V2.3/V2.3 Phase 4 文件体验与模型配置实施计划.md>)

**Goal:** 让 AI 教练中“已发送的用户消息”稳定显示附件卡片，并在刷新、切换历史会话和重新进入会话后仍能回显。

**Architecture:** 采用“消息级附件快照”方案：为 `ChatMessage` 增加 `attachments` JSON 字段，在写入 user message 时保存附件快照，并由消息读取接口直接返回。前端 `CoachTab` 在发送瞬间同时更新本地消息状态与后端请求，`MessageBubble` 只对用户消息渲染标准附件卡片。

**Tech Stack:** FastAPI / SQLAlchemy async / Alembic / pytest / httpx / React / Vite / Node test。

- [ ] **Step 4: 扩展 schema 与 API**

在 `backend/schemas/__init__.py` 中新增附件快照 schema：

```python
class ChatAttachmentSchema(BaseModel):
    model_config = ConfigDict(populate_by_name=True)

    fileId: int | None = None
    originalName: str = ""
    mimeType: str = ""
    extension: str = ""
    sizeBytes: int | None = None
```

然后让：

- `ChatMessageCreateSchema.attachments: list[ChatAttachmentSchema] = Field(default_factory=list)`
- `ChatMessageSchema.attachments: list[ChatAttachmentSchema] = Field(default_factory=list)`

在 `backend/api/chat.py`：

- `build_message_response()` 返回 `attachments=message.attachments or []`
- `append_chat_message()` 写入 `attachments=payload.attachments`
- `persist_successful_chat_turn()` 增加参数 `user_attachments`
- 增加一个内部辅助函数，根据本轮 `fileIds` 查询 `UploadedFile` 并构造消息附件快照
- user message 写入 `attachments=user_attachments or []`
- assistant message 固定写 `attachments=[]`

- [ ] **Step 3: 补消息附件快照构造逻辑**

在 `src/tabs/CoachTab.jsx` 增加一个小函数：

```javascript
function buildMessageAttachmentSnapshots(files = []) {
  return files.map((file) => ({
    fileId: Number.isInteger(file.id) ? file.id : null,
    originalName: file.originalName || file.name || '',
    mimeType: file.mimeType || '',
    extension: file.extension || '',
    sizeBytes: Number.isFinite(file.sizeBytes) ? file.sizeBytes : null,
  }))
}
```

发送时：

```javascript
const messageAttachments = buildMessageAttachmentSnapshots(attachedFiles)
const userMessage = { role: 'user', content: userInput, attachments: messageAttachments }
```

同时 `requestPayload` 增加：

```javascript
fileIds: attachedFiles.map((file) => file.id).filter(Number.isInteger)
```

恢复消息时：

- `normalizeChatMessages()` 保留 `attachments: Array.isArray(message?.attachments) ? message.attachments : []`
