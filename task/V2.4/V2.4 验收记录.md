## 成功路径

1. 用户在 AI 教练输入区上传 Markdown/TXT、DOCX、Excel 或图片文件。
2. 输入问题并发送。
3. 输入区附件 chip 清空，但该条用户消息正文上方出现附件卡片。
4. 附件卡片显示文件图标、文件名和类型/大小。
5. 刷新页面后重新进入当前会话，附件卡片仍保留。
6. 切换到其他历史会话再切回，附件卡片仍保留。
7. 若发送过程中切页触发后台思考，后台成功补回 assistant 后，原 user message 的附件快照仍保留。

## 边界情况

- 多附件：同一条用户消息可纵向堆叠多个附件卡片。
- 旧消息兼容：没有 `attachments` 的历史消息会按空数组处理，不报错也不显示空卡片。
- 删除源文件：即使 `UploadedFile` 后续被删除，历史消息仍保留当时的附件快照展示。

## 失败场景

- 上传失败文件不会进入消息级附件卡片。
- 旧 SQLite 库缺少 `chat_message.attachments` 列时，启动建表兼容逻辑会补列，避免首次发送附件消息时报错。
- 后台任务若失败，不会写入半截 assistant，也不会污染既有 user message 的附件快照。
