# Task 7 AI 对话框能力完善

- **估时**：45 分钟
- **依赖**：Task 6
- **目标**：把 AI 教练页做成一个真正像主流大模型网页端的聊天界面，参考 Claude 或 GPT 的设计，让对话框占据页面主要区域；用户看到的是一个聊天页，但模型内部注入的是健身、身体数据、训练记录等系统提示词。
- **主要文件**：
  - `src/components/CoachConversationPanel.jsx`
  - `src/tabs/CoachTab.jsx`
  - `src/utils/coachChat.js`
  - `src/api/deepseek.js`
  - `src/components/MarkdownRenderer.jsx`（如需新建）
  - `src/utils/excelImport.js`（如需新建）
- **主要操作**：
  - 让 AI 教练页面的布局更接近主流大模型网页端：聊天区域占主视觉，其他辅助信息退到侧边或次要区域。
  - 把对话框做成页面主体，用户进入这个 tab 时第一感受就是在使用一个大模型聊天界面。
  - 保留系统提示词注入、身体数据注入和训练记录注入等底层能力，但不让这些实现细节抢占视觉注意力。
  - 支持 Enter 发送、Shift+Enter 换行。
  - 增加模型切换与思考强度设置。
  - 让 AI 回复支持 Markdown 渲染。
  - 增加 Excel 文件上传与解析入口。
- **完成信号**：
  - 对话区在页面中是主区域，视觉重心明确。
  - 整体体验接近成熟的大模型聊天产品。
  - Markdown 与文件解析能在页面中正常使用。
- **验证方式**：
  - 打开 AI 教练页，确认它更像 Claude / GPT 风格的聊天页。
  - 发送多行消息、切换模型、上传 Excel，分别检查结果。
