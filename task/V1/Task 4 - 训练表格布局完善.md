# Task 4 训练表格布局完善

- **估时**：30 分钟
- **依赖**：Task 3
- **目标**：把训练计划页做成一张直观的“课表式”周训练表，既能兼容力量举周期计划，也能兼容普通增肌和自定义计划。
- **主要文件**：
  - `src/tabs/PlanTab.jsx`
  - `src/components/PlanDayCard.jsx`
  - `src/components/ExerciseEditor.jsx`
  - `src/utils/weeklyPlan.js`
- **设计思想**：
  - 兼容力量举周期计划和普通增肌 / 自定义计划。
  - 展示层直观，一眼看出 7 天训练 / 休息安排。
  - 数据结构优雅，便于 AI 快速、准确地修改训练计划。
  - 模板与实例分离，支持多种组类型和次数表达方式，动作具备主辅分级。
- **主要操作**：
  - 把周训练计划展示成课表样式：从左到右依次是周一到周日。
  - 休息日列宽更窄，训练日列宽更宽，便于用户快速扫一遍周安排。
  - 每一列内部使用多个展示卡片，一个卡片只记录一个动作及其重量、组数、次数、备注等信息。
  - 让动作展示支持主项、辅项区分，便于表达力量举周期计划与普通增肌计划的差异。
  - 让训练计划的数据结构更适合 AI 读取与改写，减少后续采纳建议时的歧义。
- **完成信号**：
  - 7 天训练 / 休息安排一眼可见。
  - 训练日与休息日的列宽和展示层级有明显差异。
  - 同一列内可容纳多个动作卡片，动作信息清晰。
  - 力量举周期计划、普通增肌计划、自定义计划都能在同一页面结构下表达。
- **验证方式**：
  - 打开训练计划页，确认页面是从左到右的 7 列课表布局。
  - 检查休息日列是否更窄、训练日列是否更宽。
  - 检查一个训练日中是否能展示多个动作卡片，且主辅分级清楚。
