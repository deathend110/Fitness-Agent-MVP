# Task 5 休息日列与空状态交互适配

- **估时**：35 分钟
- **依赖**：Task 3
- **目标**：让休息日列和空计划列适配效果稿的轻量表达方式，但不保留当前阶段无用的备注入口。
- **主要文件**：
  - `src/components/PlanDayCard.jsx`
  - `src/components/` 下休息日或空状态子组件
  - `src/utils/planLayout.js`
- **主要操作**：
  - 为休息日列设计独立模板，不再沿用训练日列的重卡片结构。
  - 处理“训练日但暂无动作”和“休息日”两类不同空状态。
  - 保证窄列在视觉上轻、但仍可点击和承载后续扩展。
  - 删除“备注入口”相关目标，避免在当前阶段引入无用交互。
- **完成信号**：
  - 休息日列明显更轻、更窄、更接近效果稿。
  - 空状态文案自然，不像坏掉的训练日列。
  - 当前休息日列不出现无实际作用的“备注入口”按钮。
- **验证方式**：
  - 检查休息日列、空训练日列的视觉差异。
  - 检查窄列是否足够轻量，同时没有无用入口残留。
