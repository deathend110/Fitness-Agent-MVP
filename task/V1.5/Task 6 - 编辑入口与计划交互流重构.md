# Task 6 编辑入口与计划交互流重构

- **估时**：65 分钟
- **依赖**：Task 4、Task 5
- **目标**：让新 UI 下的新增、编辑、删除动作流程重新变得顺手，同时不破坏现有训练计划数据写回逻辑。
- **主要文件**：
  - `src/tabs/PlanTab.jsx`
  - `src/components/PlanDayCard.jsx`
  - `src/components/ExerciseEditor.jsx`
  - `src/utils/weeklyPlan.js`
  - `src/utils/exerciseForm.js`
- **主要操作**：
  - 把“添加动作”“编辑动作”“删除动作”的入口迁移到新看板式 UI。
  - 明确新 UI 下编辑器是内联展开、抽屉、局部浮层还是其他轻量形式，并与当前结构兼容。
  - 保留原有增删改写回能力，不因为 UI 重排破坏存储。
  - 让动作卡片右上角菜单入口与实际操作联动。
- **完成信号**：
  - 新 UI 下仍能完成动作新增、编辑、删除。
  - 刷新页面后改动仍保留。
  - 删除动作不会影响其他日期，编辑入口不会误作用到错误动作。
- **验证方式**：
  - 在训练日新增一个动作并保存，刷新后确认仍存在。
  - 编辑一个百分比动作和一个固定重量动作，确认展示同步更新。
  - 删除一个动作，确认只影响当前日期。
