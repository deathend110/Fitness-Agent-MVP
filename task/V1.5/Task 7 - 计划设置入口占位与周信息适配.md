# Task 7 计划设置入口占位与周信息适配

- **估时**：55 分钟
- **依赖**：Task 2、Task 6
- **目标**：补齐效果稿中“计划设置”占位入口与周信息展示所需的轻量适配，但控制范围，不引入超出 MVP 的复杂系统。
- **主要文件**：
  - `src/tabs/PlanTab.jsx`
  - `src/utils/planLayout.js`
  - `src/utils/` 下新增训练计划页头部展示模型或周信息兼容工具
- **主要操作**：
  - 明确“计划设置”按钮的当前阶段行为：保留入口、可点击打开占位面板或提示，但不实现复杂周期计划与经典计划库本体。
  - 落地训练计划页所需的“周信息”展示口径，例如周区间、周编号等。
  - 如需要补充轻量周元信息模型，先做兼容层，再做 UI。
  - 保证旧数据仍可正常加载，不因周信息展示扩展而损坏原本计划。
- **完成信号**：
  - “计划设置”按钮存在且行为清晰，但不会误导为完整功能已上线。
  - 周区间、周编号等头部信息展示稳定。
  - 轻量扩展字段若缺失也不会导致页面异常。
- **验证方式**：
  - 点击“计划设置”入口，确认当前阶段行为与规划一致。
  - 用旧数据、缺字段数据、本周空数据分别验证周信息兼容性。
