# Task 10 今日日志页新UI适配

- **估时**：45 分钟
- **依赖**：Task 1、Task 3
- **目标**：让“今日日志”页面迁移到新 UI 风格，并与训练计划页、复杂指标面板的布局语言保持一致。
- **主要文件**：
  - `src/tabs/TodayTab.jsx`
  - `src/components/DailyMetricsPanel.jsx`
  - `src/components/WeightChart.jsx`
  - `src/index.css`
- **主要操作**：
  - 重构今日日志页的标题区、表单区、复杂指标区、体重趋势区和今日计划区。
  - 让复杂指标面板与新训练计划页的卡片密度、强调色、层级语言统一。
  - 调整两栏或多区块布局，使页面在桌面全屏下自然排布且无横向滚动。
  - 确保日志输入、复杂指标、体重趋势和今日计划四类核心信息在桌面全屏下都能完整展示。
  - 保持今日日志写回、本地复杂指标展示和 AI 跳转入口逻辑不变。
- **完成信号**：
  - 今日日志页与训练计划页属于同一套产品界面。
  - 表单、复杂指标、体重趋势、今日计划之间层级清楚。
  - 页面在全屏桌面视口下无左右滑动，只允许纵向滚动。
  - 关键区块不会因为压缩布局而只剩下局部可见信息。
- **验证方式**：
  - 录入今日日志并保存，确认功能未回归。
  - 检查复杂指标、图表和今日计划在桌面下是否压坏布局。
  - 检查全屏视口下无横向滚动。
  - 检查四类核心区块是否无需横向移动即可完整查看。
