# Task 3 周视图比例网格与列容器重构

- **估时**：50 分钟
- **依赖**：Task 2
- **目标**：把当前周计划的 7 列布局升级为效果稿中的动态比例网格，让训练日宽列、休息日窄列，并在正常 16:9 全屏视口下禁止左右滑动。
- **主要文件**：
  - `src/tabs/PlanTab.jsx`
  - `src/components/PlanDayCard.jsx`
  - `src/components/` 下新增周列壳组件
  - `src/utils/planLayout.js`
- **主要操作**：
  - 依据效果稿重构周视图网格比例，支持类似 `2fr / 1fr` 的训练日、休息日列宽差异。
  - 以“正常 16:9 全屏视口无横向滚动”为硬目标，重新约束列宽、卡片内边距和信息密度。
  - 只有在更窄的非目标视口下，才允许降级到有限兜底策略。
  - 将“列头信息”和“列内容容器”职责拆开，避免单组件继续膨胀。
  - 保证现有 weeklyPlan 数据仍能被稳定映射成列展示。
- **完成信号**：
  - 整周布局接近效果稿，看板感明显增强。
  - 训练日与休息日列宽区分稳定。
  - 在全屏 16:9 桌面视口下，训练计划主区域不出现左右滑动。
  - 页面允许纵向滚动承载更高内容密度。
- **验证方式**：
  - 在正常 16:9 全屏桌面宽度下查看 7 列布局，确认无横向滚动。
  - 检查纵向滚动时整页或内容区滚动行为是否自然。
  - 缩窄窗口，确认非目标视口下的兜底行为可接受。
  - 检查今天默认展开 / 当前列焦点逻辑是否未被破坏。
