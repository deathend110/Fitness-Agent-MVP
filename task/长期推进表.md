# RepMind V0 长期推进表

## 当前目标

按顺序完成以下阶段，并严格执行：

1. plan
2. code
3. review

执行规则：

- 每次只推进一个子任务。
- 每个子任务都使用单独的 subagent 完成实现。
- subagent 完成后立即停止，由主 agent 负责测试、验收、补充文档和提交 git。
- 每个子任务验证通过后提交一次 git，提交说明统一使用中文。
- 每个完整 Sprint 完成后再补一次阶段性中文提交。
- 如需 Python 测试，统一在本地仓库使用 `uv` 创建环境并安装依赖。
