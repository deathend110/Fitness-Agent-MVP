# Task 7 默认固定看板下的新增编辑删除能力回挂

- **估时**：70 分钟
- **依赖**：Task 6
- **目标**：在不破坏效果稿默认固定布局的前提下，把新增、编辑、删除动作能力重新挂接回训练计划页。
- **主要文件**：
  - `src/tabs/PlanTab.jsx`
  - `src/components/PlanExerciseItem.jsx`
  - `src/components/PlanExerciseEditorCard.jsx`
  - `src/components/ExerciseEditor.jsx`
  - `src/utils/planEditorState.js`
- **主要操作**：
  - 新增动作入口沿用**单日卡片最下方**的 `添加动作` 按钮，位置与样式应严格参考效果稿和当前确认图。
  - 编辑 / 删除入口沿用**单项动作卡右上角的竖向三小点**按钮，不再新增额外显式按钮行。
  - 点击竖向三小点后，出现轻量操作菜单，菜单中只包含：
    - `编辑`
    - `删除`
  - 默认看板展示层必须尽量保持和效果稿一致，交互层不要破坏动作卡片的固定排版。
  - 不再依赖展开 / 收起结构。
  - 保证写回 `weeklyPlan` 的能力不变。
  - 刷新后数据仍保留。
  - 具体交互 UI 形式参考用户提供图片与 `训练计划页面效果图.html`，优先保持“默认像效果稿，交互尽量轻量挂接”的原则。
- **完成信号**：
  - 默认态是效果稿；交互态不破坏主布局。
  - 单日底部 `添加动作` 按钮可正常触发新增动作流程。
  - 单项动作卡右上角竖向三点可打开菜单，且菜单内只有“编辑 / 删除”。
  - 新增、编辑、删除动作都能完成。
- **验证方式**：
  - 点击单日底部 `添加动作`，完成新增动作并刷新验证。
  - 点击单项动作卡右上角竖向三点，分别测试 `编辑` 与 `删除`。
  - 编辑一个动作后检查卡片展示同步更新。
  - 删除一个动作后确认只影响当前日期。
  - 刷新页面，确认新增 / 编辑 / 删除结果仍保留。
