# V1.6 效果图严格对齐长期规划表

## 范围说明

- 本规划用于承接用户新增要求：**训练计划页必须与 `训练计划页面效果图.html` 严格一致**。
- 本轮采用 **A 版方案**：不再在当前 V1.5 训练计划页上继续局部修补，而是以效果稿为唯一结构基准，重建训练计划页的展示骨架，再把现有数据与编辑能力重新挂接上去。
- 当前规划同时要求其余页面的**左侧栏 UI 与效果稿保持同一套结构与细节**，但不保留效果稿中的 `计划设置` 与 `AI 优化建议` 按钮。
- 本轮重点是**训练计划页视觉与结构对齐**，不是新增复杂周期计划、计划模板库或新的后端系统。

## 为什么需要 V1.6

虽然 V1.5 已经完成了训练计划页的新壳层、头部、网格、动作卡片和交互流第一版实现，但它仍然属于“参考效果稿”的实现，不满足当前新增标准：

1. 训练计划页和 `训练计划页面效果图.html` 仍存在明显结构差异。
2. 当前训练日列仍带有既有组件演化痕迹，不是效果稿的固定直出式结构。
3. 当前训练日还保留过展开 / 收起演化逻辑，而新要求明确不要该机制。
4. 当前侧栏壳层和效果稿侧栏在边框、留白、底部区块和按钮结构上仍不完全一致。
5. V1.5 的部分任务目标是“接近效果稿”，而当前目标已经升级为“严格按效果稿抄写元素与布局”。

因此，V1.6 不是 V1.5 的小补丁，而是一次**针对训练计划页与侧栏的严格对齐修正阶段**。

## 核心目标

V1.6 需要满足以下目标：

- 训练计划页整体结构、元素层级、间距、边框、宽窄列比例、头部工具区与效果稿严格一致。
- 训练计划页不再使用训练日展开 / 收起结构，而是固定展示 7 天看板。
- 训练计划页继续复用现有真实数据，而不是只做静态 demo。
- 动作卡片默认展示效果与效果稿一致，编辑能力改为“不打破默认布局”的挂接方式。
- 其余 tab 的左侧栏完全统一到效果稿侧栏样式，但不保留 `计划设置` 与 `AI 优化建议` 两个按钮。
- 继续遵守“正常 16:9 全屏视口不横向滚动”的约束；如果效果稿本身存在 `overflow-x-auto`，实现时应优先通过真实内容压缩与固定结构控制让目标视口下不出现横向滚动。

## 当前已知硬约束

- 继续遵守 `plan -> code -> review`。
- 代码保持 UTF-8 编码与中文注释。
- 模块超过 200 行时必须评估拆分。
- 每个子任务完成后需主 agent 验证并执行中文 git commit。
- 不回退用户已有无关改动。
- 不为了对齐效果稿而牺牲现有训练计划数据闭环。
- `计划设置` 在训练计划页内可作为训练页头部入口存在，但在其余页面左侧栏中不保留效果稿底部双按钮区。
- `AI 优化建议` 不再出现在本轮 UI 目标中。

## 设计原则

### 1. 效果稿优先

- 训练计划页的 DOM 结构、分区方式、边框和布局优先服从 `训练计划页面效果图.html`。
- 若当前代码结构与效果稿冲突，以效果稿为准，再把现有数据能力重新接入。

### 2. 默认态优先

- 用户打开训练计划页时，默认看到的就是完整 7 天固定看板。
- 不通过折叠、展开、切换模式去隐藏主布局。

### 3. 展示与编辑分离

- 默认看板展示层尽量保持和效果稿一致。
- 新增 / 编辑 / 删除能力通过轻量入口挂接，不破坏默认版式。

### 4. 侧栏统一

- 其余 tab 页面允许内部内容暂时不和训练计划页完全同构，但左侧栏必须严格统一到效果稿侧栏结构。

## 推荐执行顺序

1. 先把侧栏壳层对齐，否则训练计划页再像，整体也会割裂。
2. 再把训练计划页头部与 7 天固定骨架做成和效果稿一致的静态结构。
3. 然后逐步把训练日卡片、休息日窄列细节对齐。
4. 再把真实数据和编辑能力挂回这套固定结构。
5. 最后统一其余页面左侧栏并做整体验收。

## 风险提醒

- 当前已有 V1.5 训练计划页实现会有一部分被替换或绕过，不能把“已有实现”当作约束继续修补。
- “严格复刻效果稿”会显著提高 DOM 结构与样式的确定性要求，组件抽象过早反而容易偏离稿子。
- 训练计划页默认固定展示后，编辑能力如果直接塞回主布局，容易再次破坏效果稿外观，因此必须谨慎设计挂接方式。
- 若真实数据过多或动作名过长，可能让效果稿原始静态布局承压，需要做最小限度的文字换行与数据适配，而不是改变结构。

## 与 V1.5 的关系

- V1.5 已完成内容可作为数据层、交互层和部分基础组件参考。
- V1.6 的训练计划页实现优先级高于 V1.5 中“接近效果稿”的页面结构。
- 后续开发、验收和提交描述应明确区分“V1.5 第一版实现”和“V1.6 严格对齐修正版”。
